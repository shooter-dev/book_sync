# Résumé - Certification Développeur IA (Bloc 3)

## Vue d'ensemble

Bloc de compétences 3 : **Réaliser une application intégrant un service d'intelligence artificielle**

Compétences couvrant l'ensemble du cycle de développement d'une application IA, de l'analyse du besoin jusqu'à la livraison continue.

---

## C14 - Analyse du Besoin

### Objectif
Analyser le besoin du commanditaire et établir les spécifications fonctionnelles précises.

### Actions clés
- Rédiger les spécifications fonctionnelles
- Modéliser les données et parcours utilisateurs
- Respecter les standards d'utilisabilité et d'accessibilité
- Établir les objectifs de développement et la faisabilité technique

### Critères d'évaluation
✅ **Modélisation des données**
- Formalisme respecté (Merise, entités-relations, etc.)

✅ **Modélisation des parcours utilisateurs**
- Formalisme respecté (schéma fonctionnel, wireframes, etc.)

✅ **Spécifications fonctionnelles complètes**
- Contexte
- Scénarios d'utilisation
- Critères de validation

✅ **Accessibilité intégrée**
- Objectifs d'accessibilité dans les critères d'acceptation des user stories
- Standards appliqués : WCAG, RG2AA, etc.

---

## C15 - Conception du Cadre Technique

### Objectif
Concevoir l'architecture technique d'une application intégrant un service d'IA.

### Actions clés
- Spécifier l'architecture technique et applicative
- Préconiser les outils et méthodes de développement
- Réaliser une preuve de concept (POC)
- Favoriser les choix éco-responsables

### Critères d'évaluation
✅ **Spécifications techniques complètes**
- Architecture de l'application
- Dépendances
- Environnement d'exécution (langages, frameworks, outils)

✅ **Choix éco-responsables**
- Services (PaaS, SaaS) et prestataires éco-responsables favorisés

✅ **Flux de données**
- Diagramme de flux de données (DFD)

✅ **Preuve de concept**
- Accessible et fonctionnelle en pré-production
- Conclusion précise pour la prise de décision

---

## C16 - Coordination Agile & MLOps

### Objectif
Coordonner la réalisation technique dans un contexte agile et MLOps.

### Actions clés
- S'intégrer dans une conduite agile de projet
- Appliquer les pratiques MLOps
- Faciliter les temps de collaboration
- Atteindre les objectifs de production et de qualité

### Critères d'évaluation
✅ **Méthode agile appliquée**
- Cycles, étapes, rôles, rituels respectés
- Outils de pilotage : tableau kanban, burndown chart, backlog

✅ **Transparence et partage**
- Objectifs et modalités des rituels partagés
- Éléments de pilotage accessibles à toutes les parties prenantes

✅ **Suivi tout au long du projet**
- Respect des recommandations de la méthode choisie

---

## C17 - Développement de l'Application

### Objectif
Développer les composants techniques et les interfaces selon les spécifications.

### Actions clés
- Développer avec les langages et outils adaptés
- Respecter les spécifications fonctionnelles et techniques
- Appliquer les normes d'accessibilité, sécurité et gestion des données
- Versionner le code avec Git

### Critères d'évaluation
✅ **Environnement de développement**
- Conforme aux spécifications techniques

✅ **Interfaces**
- Intégration respectant les maquettes
- Comportements conformes aux spécifications (formulaires, animations, navigation)

✅ **Composants métier**
- Fonctionnement conforme aux spécifications

✅ **Gestion des accès**
- Droits d'accès implémentés selon les spécifications

✅ **Flux de données**
- Intégrés conformément aux spécifications

✅ **Éco-conception**
- Bonnes pratiques appliquées (éco-index, Green IT)

✅ **Sécurité**
- Top 10 OWASP implémenté

✅ **Tests**
- Tests unitaires/intégration couvrant composants métier et gestion des accès

✅ **Versionnement**
- Sources versionnées sur dépôt Git distant

✅ **Documentation technique**
- Installation de l'environnement
- Architecture applicative
- Dépendances
- Exécution des tests
- Format accessible (recommandations Valentin Haüy, Microsoft)

---

## C18 - Intégration Continue (CI)

### Objectif
Automatiser les tests lors du versionnement pour garantir la qualité technique.

### Actions clés
- Configurer un outil d'intégration continue
- Automatiser l'exécution des tests
- Versionner les configurations CI
- Documenter la chaîne CI

### Critères d'évaluation
✅ **Outil CI sélectionné**
- Cohérent avec l'environnement technique

✅ **Chaîne CI complète**
- Toutes les étapes préalables aux tests (build, configurations)
- Exécution automatique des tests

✅ **Configurations versionnées**
- Sur dépôt Git distant

✅ **Documentation CI**
- Outils, étapes, tâches, déclencheurs
- Procédure d'installation, configuration et test
- Format accessible

---

## C19 - Livraison Continue (CD)

### Objectif
Créer un processus de livraison continue pour une restitution optimale.

### Actions clés
- S'appuyer sur la chaîne CI
- Paramétrer les outils d'automatisation
- Configurer les environnements de test
- Automatiser le packaging et la livraison

### Critères d'évaluation
✅ **Chaîne CD complète**
- Toutes les étapes, tâches et déclencheurs documentés

✅ **Fichiers de configuration**
- Correctement reconnus et exécutés

✅ **Packaging automatisé**
- Compilation, minification, build de containers
- Exécution sans erreur

✅ **Étape de livraison**
- Intégrée et exécutée après validation du packaging
- Ex: pull request automatique

✅ **Sources versionnées**
- Sur dépôt Git distant

✅ **Documentation CD**
- Procédure d'installation, configuration et test
- Format accessible

---

## Modalités d'Évaluation

### E4 - Mise en situation (C14 à C19)

**Contexte** : Développement complet d'une application IA (réel ou fictif)

**Objectifs** :
- Analyser le besoin
- Concevoir l'application
- Développer l'application
- Tester l'application
- Livrer l'application

**Livrables** :
- Rapport professionnel individuel
- Soutenance orale avec démonstration du projet

**Évaluation** :
- Correction du rapport professionnel
- Présentation orale et démonstration

## Points d'Attention

### C14 - Analyse
⚠️ Ne pas oublier d'intégrer l'accessibilité dès les user stories

### C15 - Conception
⚠️ Favoriser les choix éco-responsables dans les décisions techniques

### C16 - Coordination
⚠️ Documenter et partager les rituels agiles à toutes les parties prenantes

### C17 - Développement
⚠️ Tests unitaires obligatoires pour composants métier et gestion des accès
⚠️ Documentation technique complète et accessible

### C18/C19 - CI/CD
⚠️ Versionner les fichiers de configuration CI/CD avec le code
⚠️ Documentation accessible au format recommandé
